Feature

Description

Elkhart Lake

Tiger Lake UP3

Data Streams Optimizer

Command-line tool that configures (“tunes”) I/O and processor fabric to optimize the time required to transfer data between two system endpoints, based upon specified requirements.

  • To use the tool, you need to know how data flows through the compute module, what the size of payload exchanged between the endpoints is, and the maximum tolerable latency for such data exchanges.

✓

Cache Allocation

Tools, such as the cache allocation library and cache configurator, that help bound the time needed to access data from a memory buffer based on your specified latency requirements.

  • The cache allocation library contains a set of C APIs that allocate low-latency buffers. To use the APIs in your real-time application, you should have knowledge about latency and size of the data set that your application processes, as well as the hotspots in your application’s code that are most latency sensitive.

  • Tiger Lake UP3 only: The cache configurator displays a visual representation of the cache on your system and allocates cache for use by the cache allocation library and other caching agents.

Sample application that demonstrates the basic flow of the cache allocation library. Offers command-line options and latency measurements to show the before-and-after benefit of the library.

The API Reference is a Doxygen*-generated document that provides descriptions of API elements, such as functions, data structures, and error codes. The document also shows sample application source files with links to associated descriptions.

The API Reference is available in the following locations:
