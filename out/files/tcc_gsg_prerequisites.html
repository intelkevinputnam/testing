Prerequisites

For this guide, you will need a host system and a target system that meet the following specifications.

Host System for Building Yocto Project*

The recommended way of setting up the target OS is by creating a Yocto Project-based image using the platform board support package (BSP) and Intel® TCC Tools layer integrated into it. In addition, you may choose to add the Intel® VTune™ Profiler layer to your image to experience the full capabilities of Intel® VTune™ Profiler when profiling applications on your target system.

This guide shows how to build the image for the target system. Building an image is time-consuming and requires substantial compute resources. You should ensure your host system is capable of doing it. The host system used for building the image can be different from the host system for development with Intel® TCC Tools. Consider using a dedicated server system to build the image and then share the image among developers.

To build an image, you will need a host system that meets the following specifications:

  • A Linux* distribution that supports Yocto Project.

  • Internet access.

  • A minimum of 500 GB of free disk space.

  • Recommended: Processor with a minimum of 8 cores.

  • Recommended: A minimum of 16 GB of RAM.

  • A USB drive to use as a bootable drive. You will use the drive to install the image on the target system.

    Note: When you create the bootable drive, the existing contents of the drive will be overwritten.

    • A minimum of 64 GB.

Host System for Development with Intel® TCC Tools

To develop with Intel® TCC Tools, you will need a host system that meets the following specifications:

  • Recommended: Ubuntu* 20.04 LTS

    Note:

    Instructions for Intel® TCC Tools contain steps and dependencies based on Ubuntu* 20.04 LTS as the host system. The host components of Intel® TCC Tools are expected to work on all common Linux distributions, but specific steps for setting up additional dependencies might be different.

  • Python* 3.7 or higher required for several components

  • Internet access

  • OpenSSL*

Target System

For the purposes of this get started guide, you will need one of the following boards with the specified processor installed. The processor is denoted by stock keeping unit (SKU).

Note:

In this release, the data streams optimizer works on Elkhart Lake only. The cache configurator works on Tiger Lake UP3 only.

  • Tiger Lake UP3 DDR4 customer reference board, processor SKU: QUYZ or newer

  • Elkhart Lake customer reference board, processor SKU: QTHY or newer

Additional hardware for each board:

  • Monitor

  • Keyboard

  • Mouse

  • (Optional) Ethernet cable to connect the target system to the local network for remote access

The following BIOS versions have been validated to work with this release of Intel® TCC Tools:

Note:

Other BIOS versions might work too.

  • For Tiger Lake UP3 customer reference boards:

    • TGLIFUI1.R00.3243.A04.2006302148

  • For Elkhart Lake customer reference boards:

    • EHLSFWI1.R00.2233.A07.2006180207

Note:

To acquire boards or BIOS binaries, contact your Intel representative.

Additional Prerequisites

Some samples demonstrate communication between two target systems. If you choose to run these samples, you will need additional equipment:

  • To run the RTCP (Real-Time Compute Performance) sample, you will need two Elkhart Lake target systems. Also, you will need to install an Intel® Ethernet Controller I210 on each target system (not included).

  • To run the real-time communication demo, you will need two target systems as defined above.
