Troubleshooting

Problem

Possible Cause / Solution

In Step 2: Prepare a Yocto Project*-Based Image, my host system rebooted while it was building the image. What should I do now?

To solve the problem:

  1. Follow the instructions in Restart the Build.

  2. Repeat if the system reboots again.

In Step 2: Prepare a Yocto Project*-Based Image, third-party components from meta-openembedded or poky failed with exit code 139.

Possible cause: Internet connection issue, component internal problems.

To solve the problem:

  1. Verify your Internet connection.

  2. Follow the instructions in Restart the Build.

In Step 2: Prepare a Yocto Project*-Based Image, I got an error related to checkout. Example error message:

Checkout complete

cp: cannot create regular file ...

To solve the problem:

Run the build in a new target installation directory with the command:


./tcc_tools_install_<version>.sh --keep --target <installation_directory>

In Step 2: Prepare a Yocto Project*-Based Image, I got the error:

Internal compiler error: Segmentation fault

To solve the problem:

Follow the instructions in Restart the Build.

In Step 2: Prepare a Yocto Project*-Based Image, I got the error:

Please use a locale setting which supports UTF-8 (such as LANG=en_US.UTF-8).

Python can't change the filesystem locale after loading so we need a utf-8 when python starts or things won't work.

To solve the problem:

Follow the instructions in Yocto Project.

In Step 2: Prepare a Yocto Project*-Based Image, I got the error:

ERROR:  OE-core's config sanity checker detected a potential misconfiguration.

Either fix the cause of this error or at your own risk disable the checker (see sanity.conf).

Following is the list of potential problems / advisories:

Please use a umask which allows a+rx and u+rwx

To solve the problem:

Run this command and try again: umask 022

When running tcc_integrate_bsp.sh, the console closed.

Run tcc_integrate_bsp.sh || true and analyze errors.

In Step 5: Install the Image, the target system didn’t boot from my USB drive.

To solve the problem, try the following possible solutions:

  • In the BIOS, go to Device Manager > System Setup > BootOS Selection. Change the OS from Intel Linux to Windows, or vice versa.

  • If changing the Boot OS doesn’t work, try a different USB drive.

The target system didn’t reboot when I expected it to, and pressing the power button doesn’t power on the board.

To solve the problem, disconnect the power cord, wait five seconds, and reconnect the power cord. Press the power button.

Restart the Build

To restart the build where it left off:

  1. Go to the BSP directory:

    
cd <BSP_directory>/build

  2. Source the build environment. (Note: “.” is the current directory.)

    
source ${BSP_YOCTO_FOLDER}/oe-init-build-env .

  3. To build the image:

    
bitbake multiconfig:x86:core-image-sato-sdk

  4. To build the toolchain script:

    
bitbake multiconfig:x86:core-image-sato-sdk -c populate_sdk
